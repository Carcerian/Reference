KEY DESIGN PRINCIPLES
================================================================================

1. SINGLE ENTRY POINT
   nui_mod_event.nss → routes to nui_api → calls HandleNuiEvent()

2. NO DUPLICATION
   All shared items in nui_api.nss only
   Subsystem items in their own API files

3. CLEAN SEPARATION
   Core API
     ↓
   Subsystem APIs
     ↓
   Implementations

4. NO CIRCULAR IMPORTS
   Everything flows downward only

5. MODULARITY
   Each subsystem self-contained
   Can add/remove without affecting others
